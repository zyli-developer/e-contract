# Phase 5：集成测试与上线

**目标**：端到端测试全流程、性能优化、微信小程序提审发布。

**前置依赖**：全部阶段

## 后端任务

| # | 任务 | 说明 |
|---|------|------|
| 5.1 | 全链路 E2E 测试 | 覆盖完整 MVP 流程（见测试任务） |
| 5.2 | 性能优化 | 数据库索引检查、Redis 缓存热点数据（用户信息、合同统计）、API 响应 < 200ms (P95) |
| 5.3 | 安全审计 | SQL 注入检查、XSS 防护、敏感数据加密（Token、密码）、日志脱敏（手机号） |
| 5.4 | 接口文档 | FastAPI 自动生成 OpenAPI 文档，补充业务描述和示例 |
| 5.5 | 部署配置 | Docker + Docker Compose、Nginx 反向代理、HTTPS 证书、环境变量模板 |

## 前端任务

| # | 任务 | 说明 |
|---|------|------|
| 5.6 | 体验优化 | 骨架屏（列表页/详情页）、下拉刷新动画、错误重试、网络断线提示 |
| 5.7 | 空状态设计 | 无合同、无签名、搜索无结果等空状态页面 |
| 5.8 | 兼容性测试 | 微信开发者工具 + 真机测试（iOS/Android）、基础库版本兼容 |
| 5.9 | 微信提审 | 补充隐私协议、用户协议、合规检查（用户信息获取授权弹窗）、提交审核 |

## 测试任务

| 测试 | 说明 |
|------|------|
| E2E：模板创建签署流程 | 注册 → 登录 → 创建签名 → 浏览模板 → 创建合同 → 指定签署方 → 发起签署 → 签署方登录 → 查看待签 → **短信验证码确认** → 签署 → 合同完成 → 下载 |
| E2E：文件上传签署流程 | 登录 → 上传 PDF → 指定签署方 → 发起签署 → **验证码确认** → 签署 → 完成 |
| E2E：合同取消/拒签 | 创建合同 → 发起签署 → 发起方取消 / 签署方拒签 → 状态正确 → **证据链记录完整** |
| E2E：法律合规验证 | **完成签署后 → 校验文档哈希一致 → 查询证据链包含全部 9 种操作 → 验证时间戳/IP/设备信息完整** |
| E2E：Token 安全 | Token 过期 → 自动刷新 → 请求重放；Refresh Token 重放攻击拒绝 |
| 压力测试 | 50 并发用户同时查看合同列表、Token 刷新并发安全 |
| 覆盖率检查 | 后端 ≥ 85%、前端 ≥ 75%、核心签署流程 100% |
