# 模块：用户认证与实名认证（KYC）

## 一、模块概述

用户认证模块负责平台的身份验证体系，包含账号注册登录、实名认证（KYC）、Token 生命周期管理三个子系统。所有涉及合同操作的功能均要求用户完成登录，安全签署模式还要求完成实名认证。

## 二、登录注册

### 2.1 支持的登录方式

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| 手机号 + 密码 | 传统密码登录 | 已设置密码的老用户 |
| 手机号 + 短信验证码 | 登录即注册，首次使用自动创建账号 | 新用户、忘记密码 |
| 微信授权登录 | 小程序端 `wx.login()` 静默登录 / 手机号快捷登录 | 微信小程序端 |
| 微信公众号 OAuth | H5 端 OAuth2.0 授权 | 公众号 H5 端 |

### 2.2 登录流程

```
用户输入手机号
     │
     ├── [密码登录] 输入密码 → POST /member/auth/login → 获取 Token
     │
     └── [短信登录] 发送验证码 → POST /member/auth/send-sms-code
                       │
                       └── 输入验证码 → POST /member/auth/sms-login → 获取 Token
                                                                         │
                                                                         ▼
                                                            存储 token, refreshToken,
                                                            userId, userInfo 到 localStorage
```

### 2.3 短信验证码场景

| scene 值 | 场景 |
|----------|------|
| 1 | 登录 |
| 2 | 注册 |
| 3 | 修改密码 |

### 2.4 微信授权登录

**小程序端（社交类型 34）：**
- 静默登录：`wx.login()` → 获取 code → `/member/auth/social-login` → Token
- 手机号快捷登录：`open-type="getPhoneNumber"` → 解密手机号 → 一键登录

**公众号 H5 端（社交类型 31）：**
- OAuth2.0 重定向至微信授权页 → 用户确认 → 回调携带 code → 后端换取 openid

### 2.5 相关页面

| 页面 | 路由 | 功能 |
|------|------|------|
| 登录页 | `pages/login/index` | 手机号+密码登录，跳转注册 |
| 注册页 | `pages/register/index` | 手机号+验证码登录/注册 |
| 授权回调 | `pages/login/callback` | 微信 OAuth 回调处理 |
| 修改密码 | `pages/profile/change-password` | 短信验证后修改密码 |

## 三、Token 管理机制

### 3.1 双 Token 体系

| Token | 用途 | 存储 Key | 有效期 |
|-------|------|----------|--------|
| Access Token | 业务 API 认证 | `token` | 由后端控制 |
| Refresh Token | 刷新 Access Token | `refreshToken` | 由后端控制 |
| Seal Token | 签署操作专用 | `sealToken` | 手机端 30 天，PC 端 2 小时 |

### 3.2 Token 刷新流程

```
请求发送 → 返回 401
     │
     ├── 是否正在刷新 Token？
     │       │
     │       ├── [是] 将当前请求加入等待队列
     │       └── [否] 标记刷新中 → POST /member/auth/refresh-token
     │                                    │
     │                               ┌────┴────┐
     │                               │ 成功     │ 失败
     │                               ▼         ▼
     │                          更新 Token   清除所有 Token
     │                          重放队列请求   跳转登录页
     │
     └── 队列中的请求在 Token 刷新成功后自动重发
```

### 3.3 Seal Token 管理

Seal Token 用于签署相关操作，通过 Access Token 兑换：

```
调用签署 API → 检查本地 sealToken
     │
     ├── [存在且未过期] 直接使用
     └── [不存在/已过期] POST /seal/auth/exchange-token
                              │
                              └── 缓存 sealToken（提前10分钟刷新）
```

**平台差异化 TTL（`config/seal.config.uts`）：**
- 手机端：30 天（用户常保持登录）
- PC 端：2 小时（安全性要求更高）

### 3.4 并发请求处理

当多个请求同时遇到 401 时：
1. 第一个请求触发 Token 刷新
2. 后续请求进入等待队列
3. Token 刷新成功后，队列中的请求携带新 Token 依次重发
4. 避免多次重复刷新

### 3.5 LocalStorage 存储键

| Key | 值类型 | 说明 |
|-----|--------|------|
| `token` | string | Access Token |
| `refreshToken` | string | Refresh Token |
| `userId` | string | 用户 ID |
| `userInfo` | object | 用户信息（头像、昵称等） |
| `kycStatus` | number | 实名认证状态 |
| `sealToken` | string | 签署专用 Token |
| `currentIdentity` | object | 当前身份（个人/企业） |

## 四、实名认证（KYC）

### 4.1 认证类型

| 类型 | 要求 | 适用角色 |
|------|------|---------|
| 个人实名认证 | 真实姓名 + 身份证号 + 人脸识别 | 个人用户签署合同 |
| 企业认证 | 企业名称 + 统一社会信用代码 + 营业执照 + 法人信息 | 企业身份签署合同 |

### 4.2 KYC 状态值

| 值 | 含义 |
|----|------|
| 0 | 未认证 |
| 1 | 审核中 |
| 2 | 认证成功 |
| 3 | 认证失败 |

### 4.3 个人实名认证流程

```
用户进入认证页 → 拍摄/上传身份证正反面
     │
     ├── OCR 识别（POST /member/verify/ocr/idcard）
     │       └── 自动填充姓名、身份证号
     │
     ├── 用户确认信息
     │
     ├── 人脸识别/活体检测
     │
     └── 提交认证（POST /member/verify/submit）
              │
              └── 状态变为"审核中" → 后端审核 → 成功/失败
```

### 4.4 企业认证流程

```
提供企业信息：
  - 企业名称
  - 统一社会信用代码
  - 营业执照照片
  - 法人代表姓名、身份证号
  - 法人身份证正反面照片
  - 联系人信息
     │
     └── POST /member/verify/submit-enterprise → 审核中 → 审核结果
```

### 4.5 KYC Guard 机制

系统通过 `kycGuardMixin`（`utils/kyc-check.uts`）在关键操作前拦截：

- 创建合同、签署合同、管理印章等操作前检查 `kycStatus`
- 若未认证（`kycStatus !== 2`），弹出 `KycModal` 或 `EnterpriseKycModal`
- 引导用户前往认证页面完成认证

### 4.6 相关页面

| 页面 | 路由 | 功能 |
|------|------|------|
| 个人实名 | `pages/kyc/personal/index` | 身份证 OCR + 人脸识别 |
| 企业认证 | `pages/kyc/enterprise/index` | 企业信息 + 营业执照上传 |

|------|------|------|
