# 模块：合同生命周期管理

## 一、模块概述

合同生命周期管理是平台的核心业务模块，覆盖合同从创建到归档的全流程。包含合同创建（三种方式）、签署流程（双签署模式）、合同管理（列表/筛选/操作）、合同下载与分享。

## 二、合同创建

### 2.1 创建方式

| 方式 | 入口 | 流程 |
|------|------|------|
| 模板创建 | 模板市场 → 选择模板 | 填写模板变量 → 指定签署方 → 创建 |
| AI 起草 | AI 起草首页 → 对话 | AI 多轮对话 → 生成合同 → 编辑 → 创建签署任务 |
| 文件上传 | 创建合同页 → 上传 | 上传 PDF/Word → 指定签署位置 → 指定签署方 → 创建 |

### 2.2 创建合同所需信息

```
合同基本信息：
  ├── 合同名称（必填）
  ├── 合同文件（PDF/Word/模板/AI生成）
  ├── 签署截止时间（可选）
  │
  └── 签署方列表（至少1个）：
        ├── 签署方姓名/企业名
        ├── 手机号
        ├── 签署类型：快捷签署 / 安全签署
        ├── 签署顺序
        └── 签署位置（在文档中的坐标）
```

### 2.3 创建流程（H5 集成）

合同创建的核心编辑功能通过 **WebView 嵌入 seal-core H5 页面** 实现：

```
小程序页面 (contract-create/h5-create.uvue)
     │
     └── WebView 加载 seal-core H5
              │
              ├── mode=create       → 模板创建
              ├── mode=create-file  → 文件上传创建
              └── mode=create-image → 图片上传创建
```

**H5 页面 URL 构造：**
```
{SEAL_CORE_H5_URL}/create?token={sealToken}&mode={mode}
```

## 三、合同状态机

### 3.1 状态定义

| 状态码 | 名称 | 英文 | 说明 |
|--------|------|------|------|
| 1 | 草稿 | DRAFT | 已创建未发起签署 |
| 2 | 签署中 | SIGNING | 已发起，等待签署方签署 |
| 3 | 已完成 | COMPLETED | 所有签署方已签署 |
| 4 | 已取消 | CANCELLED | 发起方取消 |
| 5 | 已拒签 | REJECTED | 签署方拒绝签署 |
| 6 | 已过期 | EXPIRED | 超过签署截止时间 |
| 7 | 填写中 | FORM_FILLING | 签署方正在填写表单字段 |

### 3.2 状态转换

```
                    ┌─────────────────────────┐
                    │                         ▼
 创建 ──→ [草稿] ──→ 发起签署 ──→ [签署中] ──→ [已完成]
            │                        │
            │                        ├──→ [已拒签]（签署方拒绝）
            │                        ├──→ [已过期]（超时）
            │                        └──→ [已取消]（发起方取消）
            │
            └──→ [已取消]（发起方删除草稿）
```

### 3.3 签署方状态

| 状态码 | 名称 | 说明 |
|--------|------|------|
| 0 | 待处理 | 等待签署方操作 |
| 1 | 已填表 | 签署方已完成表单填写 |
| 2 | 已签署 | 签署方已完成签名 |
| 3 | 已拒签 | 签署方拒绝签署 |
| 4 | 已取消 | 签署方被取消 |

## 四、合同签署

### 4.1 双签署模式

#### 安全签署（Secure Sign）

```
签署方收到通知 → 登录平台 → KYC 实名认证
     │
     ├── 查看合同详情
     ├── 填写表单（如有）
     ├── 选择印章/签名
     └── 确认签署 → 签署完成
```

**要求：**
- 必须登录平台账号
- 必须完成实名认证（KYC）
- 支持个人签名和企业印章
- 签署操作与真实身份绑定

#### 快捷签署（Quick Sign）

```
签署方收到链接 → 打开链接（无需登录）
     │
     ├── 手机号验证
     ├── 查看合同内容
     └── 手写签名 → 签署完成
```

**特点：**
- 无需注册和登录
- 手机号验证即可签署
- 仅支持个人签名，不支持企业印章
- 适合外部合作方、客户

#### 混合使用

同一份合同可以混合两种签署模式，例如：
- 发起方：安全签署（使用企业印章）
- 外部签署方：快捷签署（手写签名）

### 4.2 签署流程（H5 集成）

签署操作通过 WebView 嵌入 seal-core H5 页面完成：

```
小程序 (contract-sign/h5-sign.uvue)
     │
     └── WebView 加载签署页面
              │
              ├── 验证签署权限 (validate-permission)
              ├── 显示合同文档
              ├── 定位到签署位置
              ├── 选择/绘制签名
              └── 确认签署
```

### 4.3 签署催办

发起方可对未签署的签署方发送催办提醒：

```
POST /api/v1/seal/sign-task/{id}/urge → 发送催签通知（短信/微信消息）
```

## 五、合同管理

### 5.1 合同列表

**智能分类标签页：**

| 标签 | 筛选条件 |
|------|---------|
| 全部 | 所有合同 |
| 待填写 | 状态=填写中 |
| 签署中 | 状态=签署中 |
| 已完成 | 状态=已完成 |
| 草稿 | 状态=草稿 |
| 其他 | 已取消/已拒签/已过期 |

**筛选维度：**
- 角色筛选：我创建的 / 待我签署的 / 全部
- 状态筛选
- 关键词搜索（合同名称）
- 日期范围筛选
- 分页加载 + 下拉刷新

### 5.2 合同详情

```
合同详情页显示：
  ├── 合同基本信息（名称、状态、创建时间、截止时间）
  ├── 合同文件预览（H5 WebView）
  ├── 签署方列表（签署状态、签署时间）
  ├── 操作按钮：
  │     ├── [发起方] 催签、取消、删除
  │     ├── [签署方] 签署、拒签
  │     └── [已完成] 下载、分享、验真
  └── 操作日志时间线
```

### 5.3 合同操作

| 操作 | 角色 | 条件 | API |
|------|------|------|-----|
| 查看详情 | 所有参与方 | - | `GET /sign-task/get` |
| 签署 | 签署方 | 状态=签署中，自己待签 | 跳转签署页 |
| 催签 | 发起方 | 状态=签署中 | `POST /sign-task/{id}/urge` |
| 取消 | 发起方 | 状态=草稿/签署中 | `DELETE /sign-task/cancel` |
| 删除 | 发起方 | 状态=草稿/已取消/已拒签/已过期 | `DELETE /sign-task/delete` |
| 下载 | 所有参与方 | 状态=已完成 | `GET /sign-tasks/{id}/download` |

### 5.4 合同统计

首页展示合同统计数据：

```
GET /api/v1/seal/sign-task/statistics

返回：
  ├── 合同总数
  ├── 签署中数量
  ├── 已完成数量
  └── 草稿数量
```

## 六、合同分享

### 6.1 分享场景

| 场景 | 功能 |
|------|------|
| 查看分享 | 分享合同查看链接给他人 |
| 签署分享 | 分享签署链接给签署方（快捷签署） |
| 默认分享 | 分享小程序卡片 |

### 6.2 分享配置

```javascript
// 分享构建函数（utils/share.uts）
buildDefaultShareConfig()  // 默认分享配置
buildViewShareConfig()     // 查看分享配置
buildSignShareConfig()     // 签署分享配置
```

支持微信分享渠道：聊天、朋友圈、小程序卡片。
