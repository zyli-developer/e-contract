# 模块：合同验真与风控审查

## 一、模块概述

本模块包含两个独立子功能：
- **合同验真**：上传已签署的合同文件，验证其真实性和完整性
- **天眼审查**：AI 驱动的合同风险审查，识别潜在风险条款并提供修改建议

---

## 二、合同验真

### 2.1 功能说明

对已签署的电子合同进行多维度验证，确认合同未被篡改、签名有效、证书合规。

### 2.2 验证维度

| 维度 | 说明 |
|------|------|
| 文件完整性 | 哈希值校验，确认文件未被篡改 |
| 数字签名 | 验证签名的有效性和一致性 |
| CA 证书 | 验证签署者的 CA 证书是否合规有效 |
| 签署者信息 | 核对签署者身份信息 |
| 时间线 | 签署事件的完整时间记录 |

### 2.3 验真流程

```
用户进入合同验真页 → 上传 PDF 合同文件
     │
     ├── POST /verification/verify → 提交验证
     │
     ├── 系统解析合同文件：
     │     ├── 计算文件哈希
     │     ├── 提取数字签名
     │     ├── 验证 CA 证书链
     │     └── 解析签署者信息
     │
     └── 返回验证结果：
              ├── 验证状态：处理中 / 通过 / 未通过 / 无签名
              ├── 哈希结果：有效 / 无效
              ├── 签名结果：有效 / 无效
              ├── 证书结果：有效 / 无效
              ├── 合同信息：标题、日期、签署方数、页数
              ├── 签署者列表：姓名、类型、角色、签署时间、手机号
              └── 事件时间线
```

### 2.4 配额限制

验真功能有使用配额：
```
GET /verification/remaining → 查询剩余验真次数
```

### 2.5 相关 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/seal/verification/verify` | POST | 上传并验证合同 |
| `/api/v1/seal/verification/get` | GET | 获取验证详情 |
| `/api/v1/seal/verification/remaining` | GET | 查询剩余验真配额 |

## 三、天眼审查（AI 风控）

### 3.1 功能说明

使用 AI 对合同内容进行风险分析，自动识别风险条款并给出修改建议。

### 3.2 风险等级

| 等级 | 标识 | 说明 | 示例 |
|------|------|------|------|
| 高风险 | HIGH | 严重法律风险，建议立即修改 | 违约金比例过高、免责条款不合理 |
| 中风险 | MEDIUM | 重要风险，建议关注修改 | 付款条件模糊、知识产权归属不明 |
| 低风险 | LOW | 轻微问题，建议优化 | 格式不规范、用词不够精确 |

### 3.3 审查能力

| 能力 | 说明 |
|------|------|
| 秒级响应 | AI 快速分析，秒级返回结果 |
| 三级风险识别 | 高/中/低三级分类，优先级清晰 |
| 智能修改建议 | 针对每个风险点提供具体修改建议 |
| 精准定位 | 标注风险条款在合同中的精确位置 |

### 3.4 适用合同类型

劳动合同、销售合同、租赁合同、服务协议、供应商合同等各类合同均可使用。

### 3.5 审查流程

```
用户进入天眼审查 → 选择/上传合同
     │
     ├── 提交审查请求
     │
     ├── AI 分析合同内容
     │     ├── 条款合规性检查
     │     ├── 权利义务平衡分析
     │     ├── 违约金合理性评估
     │     └── 法律风险识别
     │
     └── 生成审查报告：
              ├── 风险总览（高/中/低数量统计）
              ├── 风险列表（按等级排序）
              │     ├── 风险描述
              │     ├── 风险位置
              │     └── 修改建议
              └── 审查结论
```

### 3.6 审查历史

用户可查看历史审查记录，跟踪审查进度。
