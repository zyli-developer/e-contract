# API 接口清单

## 一、概述

所有 API 请求通过 `api/index.uts` 中的统一 `request()` 函数发出，自动处理 Token 注入、401 刷新、错误重试等逻辑。

**基础配置（`api/config.uts`）：**

| 配置项 | 开发环境 | 生产环境 |
|--------|---------|---------|
| API Base URL | `http://localhost:48080/app-api` | `/app-api` |
| Sign H5 URL | `http://localhost:3100` | `https://h5-sign.leepm.com` |
| 默认超时 | 30,000ms | 30,000ms |

---

## 二、认证模块（auth）

**源文件：** `api/auth/login.uts`, `api/auth/social.uts`

| 接口 | 方法 | 认证 | 说明 |
|------|------|------|------|
| `/api/v1/member/auth/login` | POST | 否 | 手机号+密码登录 |
| `/api/v1/member/auth/sms-login` | POST | 否 | 短信验证码登录（登录即注册） |
| `/api/v1/member/auth/send-sms-code` | POST | 否 | 发送短信验证码 |
| `/api/v1/member/auth/refresh-token` | POST | 否 | 刷新 Access Token |
| `/api/v1/member/auth/social-login` | POST | 否 | 社交登录（微信等） |
| `/api/v1/member/auth/logout` | POST | 是 | 退出登录 |

**请求参数示例：**

```json
// 密码登录
{ "mobile": "13800138000", "password": "xxx" }

// 短信登录
{ "mobile": "13800138000", "code": "123456", "scene": 1 }

// 发送验证码
{ "mobile": "13800138000", "scene": 1 }
```

## 十一、文件上传

**源文件：** `api/config.uts`（上传配置）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/infra/file/upload` | POST | 通用文件上传 |

**上传配置：**

| 配置 | 值 |
|------|-----|
| 上传模式 | `client`（前端直传） |
| 图片大小限制 | 5 MB |
| 文档大小限制 | 20 MB |
| 图片格式 | jpg, jpeg, png, gif |
| 文档格式 | pdf, doc, docx, xls, xlsx |

---

## 十二、通用请求头

所有请求默认携带：

| Header | 值 | 说明 |
|--------|-----|------|
| `Authorization` | `Bearer {token}` | Access Token（isAuth=true 时） |
| `Content-Type` | `application/json` | 请求内容类型 |
