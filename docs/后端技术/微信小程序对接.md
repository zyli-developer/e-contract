# 后端 — 微信小程序对接指南

## 一、概述

Mini Contract.Pro 以微信小程序为主要运行平台，后端需要对接微信开放平台的多项能力。本文档详细描述后端需要实现的微信相关功能。

**微信小程序 AppID：** `wxa6f9e48551b60c87`

---

## 二、微信登录对接

### 2.1 小程序静默登录（社交类型 34）

**前端调用流程：**
```
用户打开小程序 → uni.login(provider: 'weixin') → 获取 code
     │
     └── POST /api/v1/member/auth/social-login
              Body: { type: 34, code: "0a1B2c3D...", state: "default" }
```

**后端实现步骤：**

1. **接收前端传来的 `code`**

2. **调用微信接口换取 session_key 和 openid：**
```
GET https://api.weixin.qq.com/sns/jscode2session
    ?appid={小程序AppID}
    &secret={小程序AppSecret}
    &js_code={前端传来的code}
    &grant_type=authorization_code
```

**微信返回：**
```json
{
  "openid": "oABCDEF...",
  "session_key": "key_string",
  "unionid": "unionid_if_bindable",
  "errcode": 0,
  "errmsg": "ok"
}
```

3. **根据 openid 查找或创建用户账号**
4. **生成 accessToken + refreshToken**
5. **返回登录响应（含 openid）**

> **重要：** `session_key` 必须存储在服务端，**绝不能** 返回给前端。

---

### 2.2 手机号快捷登录

**前端调用流程：**
```
用户点击"微信手机号登录"按钮 (open-type="getPhoneNumber")
     │
     ├── 获取 phoneCode（微信加密的手机号 code）
     ├── 同时调用 uni.login() 获取 loginCode
     │
     └── POST /api/v1/member/auth/weixin-mini-app-login
              Body: {
                phoneCode: "getPhoneNumber返回的code",
                loginCode: "wx.login返回的code",
                state: "default"
              }
```

**后端实现步骤：**

1. **用 `loginCode` 换取 session_key：**
```
GET https://api.weixin.qq.com/sns/jscode2session
    ?appid={AppID}&secret={AppSecret}&js_code={loginCode}&grant_type=authorization_code
```

2. **用 `phoneCode` 获取手机号（新版接口）：**
```
POST https://api.weixin.qq.com/wxa/business/getuserphonenumber
    ?access_token={接口调用凭证}
Body: { "code": "{phoneCode}" }
```

**微信返回：**
```json
{
  "errcode": 0,
  "errmsg": "ok",
  "phone_info": {
    "phoneNumber": "13800138000",
    "purePhoneNumber": "13800138000",
    "countryCode": "86",
    "watermark": { "timestamp": 1234567890, "appid": "wx..." }
  }
}
```

3. **根据手机号 + openid 查找或创建用户**
4. **返回登录响应**

> **注意：** 获取手机号需要先获取接口调用凭证 `access_token`（非用户级别，是小程序级别的）：
> ```
> GET https://api.weixin.qq.com/cgi-bin/token
>     ?grant_type=client_credential&appid={AppID}&secret={AppSecret}
> ```
> 建议缓存 access_token（有效期 2 小时），避免频繁请求。

### 2.3 社交账号绑定/解绑

**绑定：** 用户已有账号，希望关联微信

```
POST /api/v1/member/social-user/bind
Body: { type: 34, code: "wx_code", state: "default" }

后端：用 code 换取 openid → 绑定到当前登录用户
```

**解绑：** 取消微信关联

```
DELETE /api/v1/member/social-user/unbind
Body: { type: 34, openid: "oABCDEF..." }

后端：删除用户与该 openid 的关联
```

**查询：** 获取用户绑定的微信信息

```
GET /api/v1/member/social-user/get?type=34

返回：{ openid, nickname, avatar } 或 null
```

## 三、微信公众号 OAuth 对接（H5 端）

### 3.1 授权重定向

```
GET /api/v1/member/auth/social-auth-redirect?type=31&redirectUri={回调URL}
```

**后端实现：** 构建微信 OAuth2.0 授权 URL 并返回：

```
https://open.weixin.qq.com/connect/oauth2/authorize
    ?appid={公众号AppID}
    &redirect_uri={urlencode(redirectUri)}
    &response_type=code
    &scope=snsapi_userinfo
    &state={随机CSRF令牌}
    #wechat_redirect
```

### 3.2 授权回调处理

用户授权后微信重定向到回调页面，前端获取 URL 中的 `code` 和 `state` 参数，调用：

```
POST /api/v1/member/auth/social-login
Body: { type: 31, code: "回调code", state: "回调state" }
```

**后端实现：**

1. **验证 state 防 CSRF**
2. **用 code 换取 access_token + openid：**
```
GET https://api.weixin.qq.com/sns/oauth2/access_token
    ?appid={公众号AppID}&secret={公众号AppSecret}&code={code}&grant_type=authorization_code
```
3. **获取用户信息：**
```
GET https://api.weixin.qq.com/sns/userinfo
    ?access_token={上一步的token}&openid={openid}&lang=zh_CN
```
4. **查找或创建用户，返回登录响应**

## 四、JSSDK 签名（公众号 H5）

公众号 H5 页面需要调用微信 JS-SDK 能力（如分享、扫码、支付），需后端提供签名。

### 4.1 签名接口

```
POST /api/v1/member/auth/create-weixin-jsapi-signature?url={当前页面URL}
```

**后端实现步骤：**

1. **获取 jsapi_ticket：**
```
GET https://api.weixin.qq.com/cgi-bin/ticket/getticket
    ?access_token={公众号access_token}&type=jsapi
```
> `jsapi_ticket` 有效期 7200 秒，必须全局缓存。

2. **生成签名：**
```
待签名字符串 = "jsapi_ticket={ticket}&noncestr={随机字符串}&timestamp={时间戳}&url={url}"
signature = SHA1(待签名字符串)
```

3. **返回：**
```json
{
  "appId": "wxa6f9e48551b60c87",
  "timestamp": 1740000000,
  "nonceStr": "abc123random",
  "signature": "sha1_result",
  "url": "https://example.com/page"
}
```

### 4.2 前端使用的 JS-SDK 能力

| 能力 | 用途 |
|------|------|
| `chooseWXPay` | 微信支付 |
| `updateAppMessageShareData` | 自定义分享给好友 |
| `updateTimelineShareData` | 自定义分享到朋友圈 |
| `getLocation` | 获取地理位置 |
| `openLocation` | 打开地图 |
| `scanQRCode` | 扫一扫 |

## 五、微信支付对接

### 5.1 支付流程

```
前端创建订单 → POST /member/quota/order/create → 获取 orderId
     │
     ├── POST /pay/order/submit → 获取支付参数
     │       Body: { id: orderId, channelCode: "wx_lite" }
     │       Response: { displayMode: "app", displayContent: "{支付参数JSON}" }
     │
     └── 前端调用 uni.requestPayment(支付参数) → 微信支付弹窗
              │
              └── 支付完成 → POST /member/quota/order/sync-pay-status?orderId={id}
```

### 5.2 小程序支付参数

后端 `/pay/order/submit` 返回的 `displayContent` 应为微信小程序支付所需参数：

```json
{
  "timeStamp": "1740000000",
  "nonceStr": "random_string",
  "package": "prepay_id=wx20240101000000...",
  "signType": "RSA",
  "paySign": "签名字符串"
}
```

### 5.3 后端需实现

1. **统一下单接口：** 调用微信支付 V3 API 创建预付单
2. **生成支付签名：** 按微信规范生成前端支付参数
3. **支付回调处理：** 接收微信支付结果通知，更新订单状态
4. **查询支付状态：** 支持前端主动查询同步

## 六、微信消息推送（订阅消息）

### 6.1 应用场景

| 场景 | 推送内容 |
|------|---------|
| 合同待签署 | 通知签署方有新合同需签署 |
| 签署完成 | 通知发起方所有人已签署 |
| 催签提醒 | 提醒未签署方尽快签署 |
| 合同被拒签 | 通知发起方有人拒签 |

### 6.2 后端需实现

1. **申请订阅消息模板**（在微信公众平台配置）
2. **收集用户订阅授权**（前端调用 `uni.requestSubscribeMessage`）
3. **调用微信发送接口：**
```
POST https://api.weixin.qq.com/cgi-bin/message/subscribe/send
    ?access_token={access_token}
Body: {
  "touser": "用户openid",
  "template_id": "模板ID",
  "page": "pages/contract-detail/index?id=123",
  "data": {
    "thing1": { "value": "房屋租赁合同" },
    "name2": { "value": "张三" },
    "time3": { "value": "2026-02-25 10:00" }
  }
}
```

## 七、微信分享

### 7.1 小程序分享

前端通过 `onShareAppMessage` 配置分享参数，不需要后端接口。但后端需提供分享落地页数据：

```
GET /api/v1/share/summary?type=sign&targetId={taskId}
```

**返回用于分享卡片的信息：**
```json
{
  "type": "sign",
  "title": "房屋租赁合同",
  "description": "请您签署此合同",
  "creatorName": "张三",
  "statusText": "待签署",
  "expireTime": "2026-03-01T00:00:00Z",
  "requireLogin": false,
  "targetId": "task_123"
}
```

### 7.2 分享路径

| 分享类型 | 小程序页面路径 |
|---------|--------------|
| 签署分享 | `/pages/contract-sign/h5-sign?taskId=xxx` |
| 查看分享 | `/pages/contract-view/h5-view?taskId=xxx` |
| 默认分享 | `/pages/index/index` |

## 八、后端需要缓存的微信凭证

| 凭证 | 来源 | 有效期 | 用途 |
|------|------|--------|------|
| `access_token` | `/cgi-bin/token` | 2 小时 | 调用微信服务端 API |
| `jsapi_ticket` | `/cgi-bin/ticket/getticket` | 2 小时 | JSSDK 签名 |
| `session_key` | `/sns/jscode2session` | 随登录态 | 解密手机号等敏感数据 |

> **缓存策略建议：** 使用 Redis 缓存 access_token 和 jsapi_ticket，设置过期时间略小于 2 小时（如 7000 秒），避免并发刷新。

## 十、安全注意事项

1. **AppSecret 绝不能暴露给前端**
2. **session_key 必须存储在服务端**，不能传给前端
3. **OAuth state 参数** 必须校验防止 CSRF
4. **手机号解密** 必须在服务端用 session_key 完成
5. **支付签名** 必须在服务端生成
6. **access_token** 必须全局唯一缓存，防止多实例覆盖
