# 后端 — 数据模型设计

## 一、概述

以下数据模型从前端 API 调用和数据结构反向推导而来，涵盖系统核心实体及其关系。实际后端实现可根据框架和 ORM 调整字段命名和类型。

---

## 二、ER 关系总览

```
┌──────────┐     1:N     ┌──────────────────┐
│  Member   │────────────→│  SocialUser       │
│ (用户)    │             │ (社交账号绑定)      │
└────┬─────┘             └──────────────────┘
     │
     ├── 1:N ──→ Enterprise (通过 EnterpriseMember 关联)
     │              │
     │              ├── 1:N ──→ EnterpriseMember (成员)
     │              └── 1:N ──→ Seal (企业印章)
     │
     ├── 1:N ──→ Seal (个人印章/签名)
     │
     ├── 1:N ──→ SignTask (合同签署任务，发起方)
     │              │
     │              └── 1:N ──→ SignTaskParticipant (签署方)
     │
     ├── 1:1 ──→ MemberVerify (实名认证)
     │
     └── 1:N ──→ QuotaAccount (配额账户)
                    │
                    └── 1:N ──→ QuotaOrder (购买订单)
```

## 三、核心数据表

### 3.1 用户表 (member)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| mobile | varchar(20) | 手机号（唯一） |
| password | varchar(255) | 加密密码（可空，短信登录无需密码） |
| nickname | varchar(50) | 昵称 |
| avatar | varchar(500) | 头像 URL |
| status | tinyint | 状态：0=禁用, 1=正常 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |

### 3.2 社交账号绑定表 (member_social_user)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| member_id | bigint | 关联用户 ID |
| type | int | 社交类型：31=公众号, 32=开放平台, 34=小程序 |
| openid | varchar(100) | 微信 openid |
| unionid | varchar(100) | 微信 unionid（可空） |
| nickname | varchar(50) | 社交平台昵称 |
| avatar | varchar(500) | 社交平台头像 |
| create_time | datetime | 创建时间 |

**唯一索引：** `(type, openid)`

### 3.3 实名认证表 (member_verify)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| member_id | bigint | 关联用户 ID |
| real_name | varchar(50) | 真实姓名 |
| id_card | varchar(18) | 身份证号 |
| face_image_url | varchar(500) | 人脸照片 URL |
| verify_status | tinyint | 0=未认证, 1=审核中, 2=成功, 3=失败 |
| fail_reason | varchar(500) | 失败原因 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |

### 3.4 Token 表 (member_token)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| member_id | bigint | 关联用户 ID |
| access_token | varchar(500) | Access Token |
| refresh_token | varchar(500) | Refresh Token |
| access_token_expire_time | datetime | Access Token 过期时间 |
| refresh_token_expire_time | datetime | Refresh Token 过期时间 |
| refresh_token_used | tinyint | 是否已使用（一次性） |
| create_time | datetime | 创建时间 |

## 十二、索引建议

| 表 | 推荐索引 |
|----|---------|
| member | `(mobile)` UNIQUE |
| member_social_user | `(type, openid)` UNIQUE, `(member_id)` |
| enterprise | `(credit_code)` UNIQUE, `(creator_id)` |
| enterprise_member | `(enterprise_id, member_id)` UNIQUE |
| seal_info | `(member_id)`, `(enterprise_id)`, `(type, status)` |
| seal_grant | `(seal_id)`, `(enterprise_user_id)` |
| sign_task | `(creator_id)`, `(status)`, `(create_time)`, `(enterprise_id)` |
| sign_task_participant | `(task_id)`, `(member_id)`, `(mobile)` |
| quota_account | `(member_id)`, `(enterprise_id)` |
| quota_order | `(member_id)`, `(order_no)` UNIQUE |
