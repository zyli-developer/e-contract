# 后端 — 技术栈与项目结构

> **技术选型变更说明：** 原参考项目（mini-contract）后端使用 Java + Spring Boot（Yudao/RuoYi-Vue-Pro 框架）。本项目后端改用 **Python + FastAPI**，前端改用 **Taro + React**（编译为微信小程序）。所有业务需求、API 接口规范、数据模型保持不变，仅替换实现技术。

## 一、技术栈总览

| 层级 | 技术 | 说明 |
|------|------|------|
| **编程语言** | Python 3.11+ | 后端主力语言 |
| **Web 框架** | FastAPI | 异步高性能，自动生成 OpenAPI 文档 |
| **ORM** | SQLAlchemy 2.0 + Alembic | 异步 ORM + 数据库迁移 |
| **数据库** | MySQL 8.0 | 主数据库 |
| **缓存** | Redis | Token 缓存、接口限流、Session 管理 |
| **任务队列** | Celery（可选） | 异步任务（AI 生成、消息推送） |
| **包管理** | pip + requirements.txt 或 Poetry | 依赖管理 |
| **运行端口** | 8000（默认）或 48080 | 开发环境 |
| **API 前缀** | `/app-api` | 与原项目保持一致 |

## 四、FastAPI 对应 Yudao 的能力映射

原参考项目使用 Yudao 框架提供了很多开箱即用的功能。以下是用 Python 生态替代的方案：

| Yudao 功能 | Python 替代方案 | 说明 |
|-----------|----------------|------|
| Spring Security + JWT | `python-jose` + FastAPI `Depends` | Token 生成/验证 |
| MyBatis Plus ORM | SQLAlchemy 2.0 (async) | 异步 ORM，支持分页 |
| Spring Social (微信) | `wechatpy` + `httpx` | 微信小程序/公众号/支付 SDK |
| 文件上传 (OSS) | `boto3` / `python-minio` | S3 兼容协议 |
| 短信服务 | `alibabacloud-dysmsapi` / `tencentcloud-sdk-python` | 阿里云/腾讯云短信 |
| 支付模块 | `wechatpy` (微信支付 V3) | 统一下单、回调处理 |
| 定时任务/异步 | Celery + Redis | 合同过期检查、消息推送 |
| 数据库迁移 | Alembic | 类似 Flyway/Liquibase |
| API 文档 | FastAPI 自动生成 | Swagger UI + ReDoc（开箱即用） |
| 权限控制 | FastAPI `Depends` 依赖注入 | 角色权限装饰器 |
| 多租户 | 中间件 + 全局过滤 | 从请求头 `tenant-id` 解析 |

## 五、核心代码示例

### 5.1 统一响应格式

```python
# app/core/response.py
from pydantic import BaseModel
from typing import TypeVar, Generic, Optional

T = TypeVar("T")

class ApiResponse(BaseModel, Generic[T]):
    code: int = 0
    msg: str = "success"
    data: Optional[T] = None

class PageResult(BaseModel, Generic[T]):
    list: list[T] = []
    total: int = 0
    pageNo: int = 1
    pageSize: int = 10
```

### 5.2 JWT 认证

```python
# app/core/security.py
from datetime import datetime, timedelta
from jose import jwt, JWTError
from app.config import settings

def create_access_token(user_id: int, expires_hours: int = 2) -> str:
    expire = datetime.utcnow() + timedelta(hours=expires_hours)
    payload = {"sub": str(user_id), "exp": expire, "type": "access"}
    return jwt.encode(payload, settings.JWT_SECRET, algorithm="HS256")

def create_refresh_token(user_id: int, expires_hours: int = 720) -> str:
    expire = datetime.utcnow() + timedelta(hours=expires_hours)
    payload = {"sub": str(user_id), "exp": expire, "type": "refresh"}
    return jwt.encode(payload, settings.JWT_SECRET, algorithm="HS256")

def create_seal_token(user_id: int, enterprise_id: int = None,
                      expire_minutes: int = 43200) -> str:
    expire = datetime.utcnow() + timedelta(minutes=expire_minutes)
    payload = {
        "sub": str(user_id),
        "eid": enterprise_id,
        "exp": expire,
        "type": "seal"
    }
    return jwt.encode(payload, settings.JWT_SECRET, algorithm="HS256")
```

### 5.3 认证依赖注入

```python
# app/dependencies.py
from fastapi import Depends, HTTPException, Header
from fastapi.security import HTTPBearer
from app.core.security import decode_token

bearer_scheme = HTTPBearer()

async def get_current_user(
    credentials = Depends(bearer_scheme),
    db: AsyncSession = Depends(get_db)
) -> Member:
    token = credentials.credentials
    payload = decode_token(token)
    if payload.get("type") != "access":
        raise HTTPException(status_code=401, detail="Invalid token type")
    user = await db.get(Member, int(payload["sub"]))
    if not user:
        raise HTTPException(status_code=401, detail="User not found")
    return user

async def get_tenant_id(tenant_id: str = Header(alias="tenant-id", default="1")) -> str:
    return tenant_id
```

### 5.4 路由示例

```python
# app/api/v1/auth.py
from fastapi import APIRouter, Depends
from app.schemas.auth import LoginRequest, SmsLoginRequest, TokenResponse
from app.schemas.common import ApiResponse
from app.services.auth_service import AuthService

router = APIRouter(prefix="/api/v1/member/auth", tags=["认证"])

@router.post("/login", response_model=ApiResponse[TokenResponse])
async def login(req: LoginRequest, service: AuthService = Depends()):
    result = await service.login_by_password(req.mobile, req.password)
    return ApiResponse(data=result)

@router.post("/sms-login", response_model=ApiResponse[TokenResponse])
async def sms_login(req: SmsLoginRequest, service: AuthService = Depends()):
    result = await service.login_by_sms(req.mobile, req.code)
    return ApiResponse(data=result)

@router.post("/refresh-token", response_model=ApiResponse[TokenResponse])
async def refresh_token(req: RefreshTokenRequest, service: AuthService = Depends()):
    result = await service.refresh_token(req.refreshToken)
    return ApiResponse(data=result)
```

## 九、FastAPI vs Spring Boot 对照

| 维度 | Spring Boot (原方案) | FastAPI (新方案) |
|------|---------------------|-----------------|
| 语言 | Java | Python |
| 框架 | Spring Boot + Yudao | FastAPI |
| ORM | MyBatis Plus | SQLAlchemy 2.0 |
| 数据库迁移 | Flyway/Liquibase | Alembic |
| 依赖注入 | Spring DI | FastAPI `Depends` |
| API 文档 | Swagger (需配置) | 自动生成（开箱即用） |
| 异步支持 | WebFlux (较复杂) | 原生 async/await |
| 性能 | 高 | 高（异步 IO） |
| 部署 | JAR + JVM | Uvicorn + Docker |
| 学习曲线 | 较陡（Java 体系） | 较平缓（Python 语法简洁） |
