# 后端接口规范与数据结构

## 一、通用规范

### 1.1 基础配置

| 配置项 | 开发环境 | 生产环境 |
|--------|---------|---------|
| API Base URL | `http://localhost:48080/app-api` | `/app-api` |
| Seal Core H5 URL | `http://localhost:3100` | `https://h5-sign.leepm.com` |
| 默认超时 | 30,000ms | 30,000ms |
| AI 接口超时 | 180,000ms | 180,000ms |

### 1.2 统一响应格式

所有接口统一返回以下 JSON 结构：

```json
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | number | 状态码，`0` 或 `200` 表示成功 |
| `msg` | string | 提示信息 |
| `data` | any | 业务数据，可为对象、数组、字符串、null |

### 1.3 分页响应格式

分页接口统一返回：

```json
{
  "code": 0,
  "data": {
    "list": [],
    "total": 100,
    "pageNo": 1,
    "pageSize": 10
  }
}
```

### 1.4 通用请求头

| Header | 值 | 说明 | 必填 |
|--------|-----|------|------|
| `Content-Type` | `application/json` | 请求内容类型 | 是 |
| `tenant-id` | `1` | 租户 ID | 是 |
| `Authorization` | `Bearer {token}` | 用户认证令牌 | 需认证接口 |

签署业务相关接口额外使用：

| Header | 值 | 说明 |
|--------|-----|------|
| `Seal-Token` | `{sealToken}` | 签署专用 Token（注意：不是放在 Authorization 中） |

### 1.5 错误码定义

| 错误码 | 含义 | 前端处理 |
|--------|------|---------|
| `0` / `200` | 成功 | 正常处理 |
| `401` | 认证失败 | 触发 Token 刷新 |
| `403` | 授权/许可证错误 | 弹窗提示联系信息 |
| `1012005005` | Token 无效 | 触发 Token 刷新 |
| `1012005006` | Token 过期 | 触发 Token 刷新 |
| 其他 | 业务错误 | 显示 `msg` 字段内容 |

### 1.6 请求方法约定

- **GET 请求**：参数通过 Query String 传递，不使用 Request Body
- **DELETE 请求**：参数通过 Query String 传递（如 `?id=123`），不使用 Request Body
- **POST / PUT 请求**：参数通过 JSON Body 传递
