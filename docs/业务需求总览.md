# Mini Contract.Pro — 在线签合同业务需求总览

## 一、项目定位

Mini Contract.Pro 是一款面向企业和个人的**在线电子合同签署平台**，以微信小程序为主要载体，支持多端部署（微信 / 支付宝 / 百度 / 头条小程序）。平台提供从合同创建、签署、管理到归档验真的全生命周期服务。

## 二、当前阶段开发范围（MVP）

> **开发重点**：以微信小程序为载体，实现合同从创建到签署的核心闭环流程。
> 智能服务（AI 起草、天眼审查、合同验真）和企业服务（企业管理、企业印章、配额支付）**不在当前阶段范围内**，将在后续版本迭代。

### MVP 核心用户流程

```
发起方打开小程序 → 微信登录 → 创建合同（模板/上传）→ 指定签署方（手机号）→ 发送签署请求
                                                                              │
签署方收到通知 → 打开小程序 → 登录 → 在"我的合同"中看到待签合同 → 签署 → 合同生效（已完成）
```

### MVP 法律合规保障

依据《电子签名法》第十三条，电子签名需满足 4 个条件才具备与手写签名同等法律效力。MVP 采用**基础合规**方案，在不引入 KYC/CA 证书的前提下，尽可能满足法律要求：

| 合规能力 | 实现方式 | 解决的法律问题 |
|---------|---------|--------------|
| **签署意愿验证** | 签署前向签署方手机发送短信验证码，输入正确后才能执行签署 | 证明是本人自愿签署（非他人冒签） |
| **文档哈希防篡改** | 合同创建时计算 SHA-256 哈希值并存储，签署完成后再次计算存储 | 证明合同内容在签署后未被修改 |
| **签署证据链** | 记录每个关键操作：创建、发送、查看、验证码发送、验证通过、签署、完成，含时间戳/用户/IP/设备 | 完整还原签署过程，作为司法举证材料 |

**合规流程（签署方视角）**：

```
收到签署通知 → 打开合同详情 → 点击"签署" → 收到短信验证码 → 输入验证码确认
    │                                                              │
    │  ← ← ← ← ← ← ← 每一步操作均记入证据链 → → → → → → → →    │
    │                                                              ▼
    └──────────────────────────────────────────── 签署完成，文档哈希锁定
```

> **说明**：基础合规方案在大多数民事合同纠纷中可被法院采信为有效证据。如需更高法律效力（如大额交易），后续版本将引入 KYC 实名认证（身份证+人脸）和 CA 数字证书。

## 四、核心业务流程

### 4.1 合同签署主流程（MVP）

```
发起方创建合同 → 指定签署方（手机号）→ 发送签署请求
     │
     ├── 签署方收到通知（微信消息/短信）
     │       │
     │       └── 打开小程序 → 登录 → 在合同列表中看到待签合同 → 选择签名 → 签署
     │
     ├── 所有签署方完成 → 合同状态: 已完成（生效）
     ├── 任一签署方拒签 → 合同状态: 已拒签
     └── 超过截止时间    → 合同状态: 已过期
```

### 4.2 合同创建方式（MVP）

| 方式 | 流程 |
|------|------|
| 模板创建 | 浏览模板市场 → 选择模板 → 填写模板变量 → 指定签署方 → 创建 |
| 文件上传 | 上传 PDF/Word 文件 → 指定签署位置 → 指定签署方 → 创建 |

> AI 起草方式将在后续版本实现。

### 4.3 合同状态机

```
          创建
           │
           ▼
        ┌──────┐    发起签署    ┌───────┐
        │ 草稿  │──────────────→│ 签署中 │
        └──────┘               └───┬───┘
                                   │
                    ┌──────────────┼──────────────┐
                    ▼              ▼              ▼
              ┌──────────┐  ┌──────────┐  ┌──────────┐
              │ 已完成    │  │ 已拒签    │  │ 已过期    │
              └──────────┘  └──────────┘  └──────────┘

                    发起方可随时 → 已取消
```

**合同状态值：**
- `1` DRAFT — 草稿
- `2` SIGNING — 签署中
- `3` COMPLETED — 已完成
- `4` CANCELLED — 已取消
- `5` REJECTED — 已拒签
- `6` EXPIRED — 已过期
- `7` FORM_FILLING — 表单填写中

## 六、技术架构要点

> **技术选型：** 基于原参考项目（Java + uni-app）的业务逻辑，采用 Python + React 重新实现。

| 维度 | 原方案（参考） | 新方案（实施） |
|------|--------------|--------------|
| **后端语言** | Java | **Python 3.11+** |
| **后端框架** | Spring Boot (Yudao) | **FastAPI** |
| **ORM** | MyBatis Plus | **SQLAlchemy 2.0 (async)** |
| **前端框架** | uni-app + Vue 3 | **Taro 4.x + React 18** |
| **前端语言** | UTS/UVue | **TypeScript + TSX** |
| **状态管理** | localStorage 直接读写 | **Zustand** |
| **UI 组件库** | uni-ui | **NutUI-React / Taro UI** |

**保持不变的部分：**

| 维度 | 方案 |
|------|------|
| 数据库 | MySQL 8.0 + Redis 6.0 |
| API 协议 | RESTful JSON，70+ 接口，保持原路径不变 |
| 双 Token | Member Token（业务认证）+ Seal Token（签署操作） |
| H5 集成 | WebView 嵌入 Seal Core H5 页面 |
| 编译目标 | 微信小程序 + H5 |
| 主题色 | `#00C28A`（绿色） |
